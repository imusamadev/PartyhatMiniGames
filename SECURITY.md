# Security Policy

## Reporting a Vulnerability

We take the security of our users and their data seriously. If you discover a security vulnerability in this project, please report it to us as soon as possible.

### Reporting Process

1. **Email us**: Please report all security vulnerabilities to [REPLACE-WITH-YOUR-SECURITY-EMAIL] (or open a private issue if GitHub allows).
2. **Details**: Include a detailed description of the vulnerability, steps to reproduce it, and any potential impact.
3. **Response**: We will acknowledge receipt of your report within 48 hours and work with you to resolve the issue.

**Please do not report security vulnerabilities through public GitHub issues.**

## Supported Versions

Only the latest `main` branch is currently supported for security updates.
